<Lang lang="en">
    Firebase is a powerful backend solution, but without proper optimization, it can lead to performance issues and unexpected costs. Here are some tips I've learned from building production apps.

    ## Firestore Optimization

    ### Use Indexes Wisely

    Create composite indexes for complex queries. Firestore will prompt you, but proactively creating them improves query performance.

    ### Limit Query Results

    Always use `.limit()` to restrict the number of documents returned:

    ```dart
    firestore
      .collection('products')
      .where('category', isEqualTo: 'electronics')
      .limit(20)
      .get();
    ```

    ### Pagination

    Implement pagination for large datasets using `startAfter()`:

    ```dart
    final lastDocument = snapshot.docs.last;
    final nextPage = firestore
      .collection('products')
      .startAfterDocument(lastDocument)
      .limit(20)
      .get();
    ```

    ## Real-time Listeners

    ### Unsubscribe Properly

    Always cancel listeners when not needed:

    ```dart
    StreamSubscription? _subscription;

    @override
    void dispose() {
      _subscription?.cancel();
      super.dispose();
    }
    ```

    ### Use Snapshots Efficiently

    Prefer `snapshots()` over `get()` for real-time updates, but only when necessary. Each listener consumes resources.

    ## Caching Strategy

    ### Enable Offline Persistence

    For better offline experience and reduced reads:

    ```dart
    FirebaseFirestore.instance.enablePersistence();
    ```

    ### Cache Aggressively

    Cache frequently accessed data locally using packages like Hive or SharedPreferences to reduce Firestore reads.

    ## Security Rules

    Optimize security rules to prevent unnecessary reads:

    ```javascript
    match /products/{productId} {
      allow read: if request.auth != null && 
        resource.data.published == true;
    }
    ```

    ## Cost Optimization

    ### Batch Operations

    Use batch writes to reduce write operations:

    ```dart
    final batch = firestore.batch();
    batch.set(ref1, data1);
    batch.set(ref2, data2);
    await batch.commit();
    ```

    ### Monitor Usage

    Regularly check Firebase Console for usage patterns and optimize accordingly.

    ## Conclusion

    Firebase is powerful, but requires careful optimization. Focus on efficient queries, proper caching, and monitoring usage to maintain performance and control costs.
